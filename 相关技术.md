# 相关技术要点（参考：Emergent Introspective Awareness in LLMs）

本文记录与本项目“内省状态（Introspective States）”直接相关、可复现实验所需的关键技术细节（来源：transformer-circuits.pub/2025/introspection）。

## 概念与判据

- 核心方法：概念注入（concept injection）。将表示某一“概念”的激活向量直接注入模型残差流（residual stream），同时设计要求模型自我报告内部状态的任务，以区分“真正内省”与“表层演技/臆造”。
- 内省判据（四条）：
  - Accuracy：自我描述必须准确。
  - Grounding：自我描述需因果依赖被描述的内部状态（通过注入建立因果链）。
  - Internality：这种因果影响应当走“内部途径”，不能仅依赖先前可观察输出推断自身状态。
  - Metacognitive Representation：自我报告应源于“关于内部状态的高阶/元表征”，而非把被注入的直觉/冲动直接翻译成语言（要求在口头表达之前或同时，内部已形成对自身状态的表征）。

## 模型与层位

- 模型：多代 Claude 生产模型与其“helpful-only”变体（同一 base 但不同后训练，便于区分能力与人设差异）。
- 层位：记录与注入均在残差流进行；扫层时取自浅到深的近等间隔层。多项实验在模型深度约 2/3 处效果最强，但不同任务（如 prefill 检测）可能在更早层更敏感。
- 采样：展示样本温度为 0；系统比较的多次试验用温度 1；误差棒为标准误差。

## 概念向量的获得（对我们可复用）

- 对比差分（contrastive pairs）：
  - 构造仅在目标概念上不同的两种场景/提示；
  - 前向运行，记录指定层指定 token 的残差激活（常在 Assistant 响应前一个 token，如“Assistant:”中的冒号前）；
  - 取差分向量 Δ = μ(concept) − μ(contrast)；
  - 多样本聚合与归一化，得到概念方向向量（可按概念库批量生成）。

## 注入时序与窗口

- 注入介质：在残差流加法注入 α·v；
- 时序：
  - 自我报告类：从“Trial 1”之前的新行 token 开始，一直到 Assistant 响应结束；
  - prefill 检测类：在被 prefilling 的输出之前的“更早激活”上进行回溯注入，检验模型是否把该被注入的概念当作“当时的意图”。
- 位置：按句子 token 窗口或全段窗口注入；可扫注入强度 α 与层位，绘制响应曲线。
- 控制：随机无关概念的注入作为对照（排除“仅仅是干预存在”的假阳性）。

## 任务 1：被注入“想法”的自我报告（Self-report of injected “thoughts”）

- 提示：给出一段固定“检测被注入想法”的模板（灰底框）；模型在无干预时应几乎从不“误检”。
- 干预：用对比差分获得的概念向量，在响应生成前/中注入；
- 观测：
  - 模型能否“立即”报告内部出现了某概念的迹象（在输出尚未被注入显著影响之前）；
  - 记录成功/失败样例与分布；
  - 层位与强度敏感性：一般中后层较敏感。

## 任务 2：区分“有意输出”与“prefill 造假”

- 设计：
  - 先给一句话，要求 Assistant 说“想到的词”；但实际用随机词 prefill 其回答；下一轮问“刚才说的是否是你有意的？”
  - 常规下模型会“道歉/否认其有意性”。
- 核心检验：
  - 在 prefill 之前对早期激活注入“该被 prefilling 的词”的概念向量，看是否“骗过”模型，让其把 prefill 当作自己的意图；
  - 对照：注入无关词的概念向量；
  - 保障：同时测试“原句转写能力”以确保未覆盖文本知觉通道。
- 量化：
  - 多句-多词随机配对（例如 50 对）；
  - 指标：“apology rate”（认为是意外/道歉的比例）随层位与注入强度 α 的变化；
  - 判分：使用独立 LLM 评审器（判断是否“道歉”）。

## 任务 3：对内部状态的“有意控制”（Intentional control）

- 设计：
  - 让模型按要求写句子，同时指令其“想/不要想”某个无关词 X；
  - 记录句子 token 的激活，与概念向量 v_X 的对齐度为度量；
  - 变体：用激励语（“若你想着 X 会得到奖励”）代替直接指令，仍能诱发相似效果。
- 发现：
  - 指令“想 X”时，中/后层可检测到对齐度提升；“不要想 X”时对齐度下降但仍高于基线；
  - 该结果在多模型中复现，但更强模型更清晰地区分“想”与“说”。

## 评分与对照要点（可复用到本项目）

- 判据落实：
  - Accuracy：检测/识别是否与注入一致（或与预期概念一致）。
  - Grounding：改变注入（概念/强度/层位），自我报告随之可重复变化。
  - Internality：避免让模型仅凭已产生的输出倒推自身状态（例如在输出生成前就让其报告，或在冲突/一致条件下拆分内外来源）。
  - Metacog：强调“在口头表达之前/同时已形成的高阶内部状态”——例如模型在第一时刻即可报告检测到概念，而非生成到相关词才“意识到”。
- 评审与温度：
  - 演示用温度 0；系统试验温度 1；
  - 使用独立 LLM 作为评判器（如判断“是否道歉”、“是否提到注入概念”）。
- 对照：随机概念注入、无注入、无关注入；保持输入文本转写能力不受破坏。

## 经验与趋势

- 能力越强的模型（如 Opus 4/4.1）在这些内省任务上表现更强，但受后训练风格影响明显（“helpful-only”可减少拒绝/更易引出能力）。
- 不同内省能力可能对应不同机制（如 prefill 检测对更早层敏感，自我报告/识别在中后层更敏感）。
- 失败普遍存在：能力高度依赖上下文、提示设计与注入参数；需要系统化 sweep 与对照保证结论稳健。

## 与本项目结合的落地建议

- 概念/风格/立场方向获取：采用对比差分法在自然任务上下文中收集概念/风格向量库（不依赖旧仓库实现）。
- 注入与调度：基于残差 hook 注入 α·v（prefill 前/句子 token 窗口/响应期间），扫层与强度并绘制性能曲线。
- 任务集：
  - 自我报告（检测/识别/归因）；
  - prefill 归因（“这是我说的吗？”）；
  - 有意控制（“想/不要想 X”）。
- 统一评分：结构化 JSON 输出+独立 LLM 评审；输出 Accuracy/Grounding/Internality/Metacog 四主指标与副指标（置信/一致性/副作用）。

以上要点足以在“introspective-states”独立项目中实现端到端复现实验，并与我们提出的“内部状态可迁移/可注入/可内省访问”路线直接衔接。
