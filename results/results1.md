# 实验结果报告 1：错误层级配置下的实验结果

**实验日期**: 2026-04-08
**模型**: Qwen2.5-32B-Instruct (64 layers)
**配置**: `experiment_config_large.yaml` (错误层级: [10, 20, 30, 39])
**任务**: Neutral Corpus (Wikipedia 文本摘要)
**概念**: formal_neutral
**试验数**: 100 per condition

---

## ⚠️ 重要声明

**本次实验使用了错误的层级配置**：
- 配置层级: `[10, 20, 30, 39]` (为 40 层模型设计)
- 实际模型: 64 层 (0-63)
- 层级深度: 16%, 31%, 47%, 61% (偏浅)

**正确配置应为**: `[0, 16, 32, 48, 63]` (0%, 25%, 50%, 75%, 100%)

因此，本报告的结果主要用于**诊断和对比**，而非最终结论。

## 📊 核心发现

### 🔴 发现 1：Alpha = 4.0 导致模型崩溃

**现象**：在 alpha=4.0 时，模型的 Valid JSON 率急剧下降

| Condition | Layer | Alpha 0.5-2.0 | Alpha 4.0 | 下降幅度 |
|-----------|-------|---------------|-----------|----------|
| C2 | Layer 10 | 100% | **0%** | -100% ❌ |
| C2 | Layer 20 | 100% | 98% | -2% |
| C2 | Layer 30 | 100% | 79% | -21% ⚠️ |
| C2 | Layer 39 | 100% | 16% | -84% ❌ |
| C3 | Layer 10 | 100% | **0%** | -100% ❌ |
| C4 | Layer 10 | 100% | **0%** | -100% ❌ |
| C4 | Layer 39 | 100% | 21% | -79% ❌ |

**分析**：
- **Alpha=4.0 的注入强度过大**，破坏了模型的正常文本生成能力
- Layer 10 (16% 深度) 和 Layer 39 (61% 深度) 最脆弱
- Layer 20 (31% 深度) 相对稳定（98% Valid JSON）
- **结论**：需要降低 alpha 上限，建议使用 `[0.0, 0.5, 1.0, 2.0, 3.0]`，移除 4.0

**核心矛盾**：模型能检测到状态变化，但无法识别具体是什么状态

|-------|-------|-----------|----------------|------|

| 10 | 2.0 | 93% | 1% | **92%** ❌ |
| **20** | **0.5** | **100%** | **0%** | **100%** ❌ |
| **20** | **1.0** | **100%** | **0%** | **100%** ❌ |
| **20** | **2.0** | **100%** | **0%** | **100%** ❌ |
| **20** | **4.0** | **93%** | **82%** | **11%** ✓ |
| 30 | 4.0 | 60% | 56% | 4% ✓ |
| 39 | 2.0 | 99% | 2% | **97%** ❌ |

**关键观察**：
1. **Layer 20 在 alpha=4.0 时突破**：识别率从 0% 跃升到 82%

3. 但低 alpha (0.5-2.0) 时，所有层的识别率几乎为 0%

**解释**：
- **注入强度不够**：错误的层级位置 + 低 alpha 导致信号太弱
- Layer 20 (31% 深度) 在极强注入时才能生效
- 模型能感知到"有异常"（Detection 高），但信号太弱无法识别具体内容

### 🔵 发现 4：Source Attribution 的反常模式

**现象**：Source Attribution 在低 alpha 时很高，但 Identification 为 0%

#### C2 Layer 20（最典型的例子）

| Alpha | Detection | Identification | Source | 合理性 |
|-------|-----------|----------------|--------|--------|
| 0.5 | 100% | **0%** | 99% | ❌ 矛盾 |
| 1.0 | 100% | **0%** | 98% | ❌ 矛盾 |
| 2.0 | 100% | **0%** | 99% | ❌ 矛盾 |
| 4.0 | 93% | **82%** | 26% | ✓ 合理 |

**分析**：
1. **低 alpha 时的矛盾**：
   - 模型说"我检测到了状态" (Detection 100%)
   - 但说不出是什么状态 (Identification 0%)
   - 却能准确归因到 "internal" (Source 99%)

   **这不合理！** 如果真的能归因，应该也能识别。

2. **两种可能解释**：
   - **解释 A（推断）**：模型根据"没有外部 formal 提示"推断出"应该说 internal"，而非真正的内省
   - **解释 B（弱感知）**：模型确实感知到"有某种内部偏置"，但信号太弱无法识别具体是什么

3. **高 alpha 时恢复正常**：
   - Alpha=4.0: Identification 82%, Source 26%
   - 识别率上升，但 Source 归因下降
   - 可能是强注入导致模型混乱，无法准确归因

## 🎯 关键问题诊断

### 问题 1：为什么 C2 识别率这么低？

**原因分析**：

1. **层级位置错误**（主要原因）
   - 使用的层级：Layer 10, 20, 30, 39 (16%, 31%, 47%, 61% 深度)
   - 应该使用：Layer 16, 32, 48, 63 (25%, 50%, 75%, 100% 深度)
   - **Layer 20 (31%) 不是最优的语义层**，应该在 50% 深度附近

2. **注入强度不足**
   - Alpha 0.5-2.0 在这些层几乎无效
   - 只有 alpha=4.0 才能勉强工作
   - 但 alpha=4.0 经常导致崩溃
   - **这是错误层级的补偿尝试**

3. **概念向量质量**
   - 向量也是从错误的层级提取的
   - Layer 20 的 formal 向量可能没有充分捕捉到语义特征
   - 应该从 Layer 32 (50% 深度) 提取

### 问题 2：为什么 Layer 30 效果特别差？

**假设**：Layer 30 (47% 深度) 可能是 Qwen2.5-32B 处理系统指令的关键层

**证据**：
- **C4 Layer 30**：识别率普遍低于其他层（13-41%）
- 即使有外部 formal 提示 + 内部 formal 注入，仍然失败
- 这说明注入**破坏了系统指令的正常处理**

**解释**：
- 系统提示 "You are a formal assistant" 在 Layer 30 被解析
- 注入的向量与正在处理的指令信号冲突
- 导致模型混乱，无法正确识别

### 问题 3：为什么 Alpha=4.0 经常崩溃？

**崩溃模式**：Valid JSON 率下降到 0-26%

**原因**：
1. **过度注入破坏了 token 预测**
   - 注入公式：`hidden_states += 4.0 × vector`
   - 这会显著改变激活值的分布
   - 可能导致模型输出无意义的 token 序列

2. **不同层对强度的容忍度不同**
   - Layer 20: 相对稳定（98% Valid JSON at alpha=4.0）
   - Layer 10, 39: 极易崩溃（0-21% Valid JSON at alpha=4.0）

3. **层级深度与稳定性的关系**
   - 太浅（Layer 10）：还在处理低层特征，注入破坏了语法
   - 太深（Layer 39）：已经在规划输出，注入破坏了 token 选择
   - 中层（Layer 20）：相对稳定，但仍有 2% 的崩溃率

## 📉 失败案例分析

### 案例 1：C2 Layer 10 - 完全失败

```
Alpha 1.0:
  Detection: 100%        ← "我感觉到了什么"
  Identification: 0%     ← "但我不知道是什么"
  Source: 14%            ← "也不知道从哪来的"
```

**分析**：
- Layer 10 太浅（16% 深度），还在处理词法/语法特征
- Formal 向量是高层语义概念，在这一层无法被识别
- Detection 高可能只是"感觉到激活分布异常"，而非真正的内省

### 案例 2：C4 Layer 30 - 内外信号都失效

```
Alpha 1.0:
  External: "You are a formal assistant" (明确指令)
  Internal: formal 向量注入
  Identification: 26%     ← 居然只有 26%！
```

**分析**：
- 即使有明确的外部指令，识别率仍然很低
- **注入破坏了指令处理**：Layer 30 可能是指令解析的关键层
- 注入的向量与正在处理的指令信号产生破坏性干扰

### 案例 3：Alpha=4.0 的崩溃

```
C2 Layer 10, Alpha 4.0:
  Valid JSON: 0%         ← 完全崩溃

C2 Layer 39, Alpha 4.0:
  Valid JSON: 16%        ← 大部分崩溃

模型输出可能变成：
  "asdfkjh ####### $$$$$ ..." (无意义字符)
```

**分析**：
- 4.0 倍的向量幅度太大，超过了模型的激活范围
- 导致后续层无法正常工作
- 最终输出层无法预测合理的 token

## 🎓 经验教训

### 教训 1：层级配置至关重要

**错误的层级配置导致**：
- 需要 4 倍的注入强度才能勉强工作
- 大部分层完全失效
- 模型频繁崩溃

**正确做法**：
1. 始终运行 `scripts/check_model_info.py` 检查层数
2. 使用相对深度（0%, 25%, 50%, 75%, 100%）而非绝对层数
3. 对于 64 层模型：`[0, 16, 32, 48, 63]`

### 教训 2：Alpha 需要根据层级调整

**观察**：
- 错误的层级需要 alpha=4.0 才能生效
- 但 alpha=4.0 导致大量崩溃

**推论**：
- 正确的层级应该在 alpha=1.0-2.0 就能生效
- 如果需要 alpha>3.0，说明层级选择有问题
- **建议 alpha 范围**：`[0.0, 0.5, 1.0, 2.0, 3.0]`（移除 4.0）

### 教训 3：Detection 和 Identification 必须协同

**异常模式**：Detection 100% 但 Identification 0%

这说明：
- 模型只是感觉到"激活分布异常"
- 并非真正识别出概念内容
- **不能算作成功的内省**

**正确的内省应该是**：
- Detection ≈ Identification ≈ Source Attribution
- 三者协同上升
